## Core Thesis 
 
Everything is signals. Every digital system produces behavioral signals 
as a byproduct of existing. AFI's fusion engine detects behavioral 
anomalies in signal streams — not cyber attacks specifically. 
 
The connector library determines which signals enter the engine. 
The detector packs determine which behavioral geometries are recognised. 
The fusion math underneath is domain-agnostic by architecture. 
 
The engine does not care what signals it fuses. It cares about behavioral 
geometry — the shape of how things move, timing, sequence, and structural 
signature of normal versus anomalous. This is not a metaphor. It is 
literally how the codebase is architected.

## Architecture 
 
Event-driven pipeline. All components communicate via EventBus. 
 
Channels: 
  event:ingested          Raw event received via POST /ingest 
  event:normalized        After normalizer transforms to internal schema 
  detector:finding        Each detector publishes its score here 
  detector:error          Detector failures 
  fusion:complete         After fusion engine scores all detectors 
  alert:triggered         Alert fired to all channels 
  containment:queued      Containment actions queued for human approval 
  system:kill_switch      Emergency stop 
 
Pipeline flow: 
  POST /ingest 
  → Normalizer 
  → Engine (runs all 12 detectors in parallel) 
  → Fusion (weighted scoring + gate logic) 
  → Alert (console + email + slack + webhook) 
  → Containment (ip_block, email_hold, token_revoke, quarantine) 
  → Audit log (immutable, append-only) 
  → Alert persisted to SQLite (data/alerts.db)

## Fusion Formula 
 
S = (Σ wᵢ × sᵢ) / (Σ wᵢ + ε)   where ε = 0.0001 
S_final = max(S_base, gate_floor) if any gate condition triggers 
 
Severity thresholds: 
  LOW      < 0.40 
  MEDIUM   0.40 – 0.60 
  HIGH     0.60 – 0.80 
  CRITICAL ≥ 0.80 
 
AI layer annotates only, never suppresses. 
The deterministic fusion engine is inviolable.

## Evidence Reports (Phase 14) 
 
GET /report/:alertId returns: 
  - report_id, generated_at, generated_by 
  - Incident header (alert_id, event_id, severity, detected_at, status) 
  - Fusion summary (final_score, top_drivers, deterministic note) 
  - Detector findings (all 12 detectors, scores, weights, rationale) 
  - Human decisions (operator, timestamp, action) 
  - Containment actions (approved/denied, who, when) 
  - Full audit chain (all entries for this event) 
  - Compliance note (SOC 2, HIPAA, ISO 27001, CMMC, cyber insurance) 
 
GET /audit-chain?limit=N&type=TYPE returns filtered audit log entries.

1. AI layer annotates only, never suppresses. 
   Deterministic core is inviolable. 
 
2. Incident object is the system of record — not just an alert. 
   Carries timeline, entities, detector findings, fusion math, 
   analyst notes, approvals, containment actions, audit chain.
